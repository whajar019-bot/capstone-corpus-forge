# Prompt History

This file documents all prompts used with VS Code Copilot during development.

## database.py
Promt N.1 - I am building a Flask web application called Corpus Forge for my EPITA university capstone project. It is an AI-powered document knowledge platform. I need you to write the contents of a file called database.py. This file manages a SQLite database called corpus_forge.db. It needs three tables: 1) documents table with columns: id (primary key), filename (text), filetype (text), content (full text of the document), upload_date (text), is_active (integer, 0 or 1). 2) artifacts table with columns: id (primary key), type (text, can be 'flashcard', 'quiz', or 'code_review'), content (text, stored as JSON string), document_id (integer, foreign key to documents), created_date (text). 3) usage table with columns: id (primary key), request_count (integer), token_count (integer), timestamp (text). Write functions to: insert a document, get all documents, get active documents, toggle a document active or inactive, delete a document, insert an artifact, get all artifacts by type, insert a usage record, and get total usage counts. Use only the sqlite3 library which comes built into Python, no other libraries needed, please.

Prompt N.2 - I am building a Flask web application called Corpus Forge for my EPITA university capstone project. Write a Python file called ingestor.py that extracts text from three file types: 1 PDF files using the PyMuPDF library (imported as fitz), extract all text from all pages. 2 Plain text and markdown files (.txt, .md), just read the file content as UTF-8 text. 3 Source code files (.py, .js), read the file content as UTF-8 text. Write a main function called ingest_file(filepath) that takes a file path as a string, detects the file type from its extension, calls the correct parser, and returns the extracted text as a string. If the file type is not supported, raise a ValueError with a clear message.

Promt N.3 - I am building a Flask web application called Corpus Forge for my EPITA university capstone project. Write a Python file called retriever.py that uses ChromaDB for document storage and retrieval. It needs these functions: 1 init_chroma() that initializes a local ChromaDB client and creates a collection called 'corpus'. 2 add_document(doc_id, text, filename) that takes a document's text, splits it into chunks of 500 characters with 50 character overlap, and adds each chunk to ChromaDB with the doc_id and filename as metadata. 3 query(question, doc_ids, n_results=3) that takes a question and a list of active document ids, searches ChromaDB for the most relevant chunks from only those documents, and returns the top results as a single combined string. 4 delete_document(doc_id) that removes all chunks belonging to a document from ChromaDB. Use only the chromadb library.
