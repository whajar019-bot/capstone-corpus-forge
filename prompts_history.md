# Prompt History

This file documents all prompts used with VS Code Copilot during development.

## database.py
Promt N.1 - I am building a Flask web application called Corpus Forge for my EPITA university capstone project. It is an AI-powered document knowledge platform. I need you to write the contents of a file called database.py. This file manages a SQLite database called corpus_forge.db. It needs three tables: 1) documents table with columns: id (primary key), filename (text), filetype (text), content (full text of the document), upload_date (text), is_active (integer, 0 or 1). 2) artifacts table with columns: id (primary key), type (text, can be 'flashcard', 'quiz', or 'code_review'), content (text, stored as JSON string), document_id (integer, foreign key to documents), created_date (text). 3) usage table with columns: id (primary key), request_count (integer), token_count (integer), timestamp (text). Write functions to: insert a document, get all documents, get active documents, toggle a document active or inactive, delete a document, insert an artifact, get all artifacts by type, insert a usage record, and get total usage counts. Use only the sqlite3 library which comes built into Python, no other libraries needed, please.

Prompt N.2 - I am building a Flask web application called Corpus Forge for my EPITA university capstone project. Write a Python file called ingestor.py that extracts text from three file types: 1 PDF files using the PyMuPDF library (imported as fitz), extract all text from all pages. 2 Plain text and markdown files (.txt, .md), just read the file content as UTF-8 text. 3 Source code files (.py, .js), read the file content as UTF-8 text. Write a main function called ingest_file(filepath) that takes a file path as a string, detects the file type from its extension, calls the correct parser, and returns the extracted text as a string. If the file type is not supported, raise a ValueError with a clear message.

Promt N.3 - I am building a Flask web application called Corpus Forge for my EPITA university capstone project. Write a Python file called retriever.py that uses ChromaDB for document storage and retrieval. It needs these functions: 1 init_chroma() that initializes a local ChromaDB client and creates a collection called 'corpus'. 2 add_document(doc_id, text, filename) that takes a document's text, splits it into chunks of 500 characters with 50 character overlap, and adds each chunk to ChromaDB with the doc_id and filename as metadata. 3 query(question, doc_ids, n_results=3) that takes a question and a list of active document ids, searches ChromaDB for the most relevant chunks from only those documents, and returns the top results as a single combined string. 4 delete_document(doc_id) that removes all chunks belonging to a document from ChromaDB. Use only the chromadb library.

Prompr N.5 - Create a Python file called ai_client.py that uses the google-generativeai library to interact with Google Gemini. It needs these functions: 1) init_client() that reads the GOOGLE_API_KEY from environment variables using python-dotenv and initializes the Gemini client using the gemini-1.5-flash model. 2) chat(context, question, audience, tone) that takes retrieved document context, a user question, audience level (beginner/intermediate/expert) and tone (formal/casual), builds a prompt, sends it to Gemini, and returns the answer as a string. 3) generate_flashcards(context, count) that generates flashcards from the context and returns a JSON list of objects with 'front' and 'back' keys. 4) generate_quiz(context, count) that generates multiple choice questions and returns a JSON list of objects with 'question', 'options' (list of 4), and 'answer' keys. 5) review_code(code_text) that generates a code review report and returns it as a string. 6) get_usage() that returns a dict with total_requests and total_tokens counted across all function calls. Track usage after every API call using the response usage metadata.

Prompt N.6 - Create a Python file called app.py for a Flask web application called Corpus Forge. It imports and uses four modules: database, ingestor, retriever, and ai_client. It needs these routes: 1) GET / that serves templates/index.html. 2) POST /upload that accepts a file upload, saves it to the uploads folder, calls ingestor.ingest_file() to extract text, calls retriever.add_document() to store in ChromaDB, calls database.insert_document() to save to SQLite, returns JSON with the new document info. 3) GET /documents that returns JSON list of all documents from database. 4) POST /documents/<id>/toggle that toggles a document active or inactive in the database. 5) DELETE /documents/<id> that deletes document from database, ChromaDB, and uploads folder. 6) POST /chat that accepts JSON with question, doc_ids list, audience and tone, calls retriever.query() to get context, calls ai_client.chat() and returns the answer as JSON. 7) POST /flashcards that accepts JSON with doc_ids and count, gets context, generates flashcards, saves to database as artifact, returns JSON flashcards. 8) POST /quiz that accepts JSON with doc_ids and count, gets context, generates quiz, saves to database as artifact, returns JSON quiz. 9) POST /code-review that accepts JSON with doc_ids, gets context, generates code review, saves to database as artifact, returns the report as JSON. 10) GET /usage that returns total request count and token usage from ai_client.get_usage(). Call database.init_database(), retriever.init_chroma(), and ai_client.init_client() when the app starts.

Use only plain HTML CSS and JavaScript with no external frameworks. Use fetch() to call the Flask API.
